A small language built for clarity

Static types, algebraic data types, and a sym CLI — tree interpreter today, optional bytecode VM for a compileable subset. Designed to be easy for humans, editors, and models to reason about.

What you get

Type system

Int, Bool, String, Unit, ADTs with type / match, prelude Option and List.

Two runtimes

Default tree-walking interpreter; sym run --vm for a stack VM on a well-defined subset.

Modules

import with stitched sources and # sym:file markers for diagnostics across files.

Built-ins

I/O, strings, JSON helpers, HTTPS (http_post, SSE fold), env, filesystem — see contracts in-repo.

Tooling

Human-readable or single-line JSON errors; conformance tests for VM vs interpreter parity.

Open source

Rust implementation, CI on Ubuntu & macOS, contributing & security guidelines on GitHub.

Install & run

Requires Rust stable (MSRV 1.74). Clone the repo, then: 

cargo build -p symc --release
./target/release/sym run examples/hello.sym

# or during development
cargo run -p symc -- run examples/hello.sym

Install the binary crate locally: cargo install --path crates/symc

Useful commands

sym check FILE.sym      # typecheck + imports
sym run FILE.sym        # run main
sym run --vm FILE.sym   # bytecode VM (subset)
sym tokens FILE.sym     # lexer debug

JSON diagnostics: sym check --message-format json …

Minimal example

fn greet(name: String) -> Unit =
  println(concat("Hello, ", concat(name, "!")))
end

fn main() -> Unit =
  greet("Sym")
end
