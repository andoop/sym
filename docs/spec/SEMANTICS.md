# Sym 操作语义（初稿）

与当前 `symc` 实现一致；与 [LANGUAGE.md](./LANGUAGE.md) 配套阅读。

## 1. 环境

- **全局**：所有顶层 `fn` 名在求值前绑定为 `FnRef(name)`。
- **局部**：`let`、`do` 块、`match` 分支引入绑定；内层遮蔽外层；按栈式帧实现（`EvalEnv`）。

## 2. 表达式求值顺序

| 形式 | 顺序 |
|------|------|
| `f(a,b,…)` | 求值 `a,b,…` 左到右，再应用 `f`（内建或用户函数）。 |
| `if c then t else e` | 先 `c`；仅求值被选分支。 |
| `while c do b` | 反复：求 `c`；若为真求 `b` 并丢弃结果；否则结束，整体为 `()`。 |
| `l && r` | 先 `l`；仅当为真时求 `r`（字节码与树解释器均短路）。 |
| `l \|\| r` | 先 `l`；仅当为假时求 `r`。 |
| `let x = v in e` | 先 `v`，再在扩展环境中求 `e`。 |
| `do s…; t end` | 顺序执行语句，再求值尾部 `t`。 |

## 3. 算术与比较

- **整数**：`+ - *` 通常语义；`/` 向零截断；`%` 为欧几里得余数（`rem_euclid`）。
- **`==` / `!=`**：按 `Value` 的相等性；类型错误在类型检查阶段排除。
- **有序比较**：仅 `Int` 与 `Int`、`String` 与 `String`；字符串按 **Unicode 标量值** 字典序。

## 4. `match`

- 按臂顺序尝试模式；首个成功匹配的执行其体。
- 穷尽性由类型检查器保证；运行时若仍落入无匹配，为解释器错误。

## 5. 发散（`Never`）

- `exit(n)` 等不返回；与 `Unit` 等类型的合一规则见类型检查器（用于 `if` 两分支类型合并等）。

## 6. 拼接源与诊断坐标

- 类型检查与解析的 `Span` 相对于 **拼接后** 缓冲区。
- JSON 诊断中的 **`logical_file` / `logical_line` / `logical_column`**：在含 `# sym:file` 的缓冲区内，表示该错误位置在 **对应片段正文**（标记行之后、下一文件标记之前）内的路径与行列；**不等于** 磁盘上原始 `.sym` 行号（因剔除 `import`/`module` 行与 `trim` 等），后续可增强为精确映射。

## 7. VM 子集

- 仅保证上述语义在 **可编译** 的子集上由字节码 VM 复现；未支持构造在编译期拒绝。
