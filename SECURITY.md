# Security policy

## Supported versions

We treat the **`main` branch** and the latest published state of this repository as in scope. There is no separate LTS line yet.

## Reporting a vulnerability

**Please do not** open a public GitHub issue for undisclosed security problems.

Preferred options:

1. **[GitHub Security Advisories](https://github.com/andoop/sym/security/advisories/new)** for this repository (private report to maintainers).
2. If you cannot use GitHub: contact the repository owner via the email on their [GitHub profile](https://github.com/andoop), with subject line containing **`[symc-security]`**.

Include:

- Description of the issue and impact
- Steps to reproduce (minimal `.sym` / CLI invocation if applicable)
- Affected component (interpreter, VM, I/O builtins, etc.)

We will try to acknowledge within a few days; please allow time for triage and a fix before public disclosure.

# 安全披露（中文）

## 支持范围

当前以本仓库 **`main` 分支**及最新代码为准；尚无单独 LTS 线。

## 如何报告

**请勿**在公开 issue 中讨论未修复的漏洞细节。

推荐方式：

1. 使用本仓库的 **[GitHub Security Advisories](https://github.com/andoop/sym/security/advisories/new)** 私密提交。
2. 若无法使用上述功能：通过维护者在 [GitHub 主页](https://github.com/andoop) 公开的邮箱联系，邮件主题请包含 **`[symc-security]`**。

请尽量提供：现象与影响、复现步骤（如有请附最小 `.sym` 与命令行）、涉及模块（解释器 / VM / 内建 I/O 等）。

我们会在数日内尽量回复；在修复发布前请配合保密。
